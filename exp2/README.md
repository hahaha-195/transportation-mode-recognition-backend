# Exp2: 轨迹+基础知识图谱

## 实验概述

Exp2在Exp1的基础上添加基础知识图谱特征，验证地理语义信息对交通方式识别的价值。该实验的目标是证明多源数据融合的有效性，特别是知识图谱特征对识别性能的提升作用。

## 实验设计

### 特征组合

- **轨迹特征**: 9维
  - speed（速度）
  - acceleration（加速度）
  - distance（累计距离）
  - bearing（方向角）
  - bearing_change（方向变化）
  - speed_change（速度变化）
  - time_interval（时间间隔）
  - distance_change（距离变化）
  - duration（持续时间）

- **基础知识图谱特征**: 11维
  - 道路类型（primary/secondary/residential等）
  - POI距离（公交站/地铁站距离）
  - 空间语义（是否在市区/郊区）
  - 交通网络（道路连通性）

### 融合策略

- **硬拼接**: 轨迹+KG独立编码后拼接
- **决策层**: 所有特征直接影响分类决策

## 模型架构

### Bi-LSTM模型（多输入）

```
轨迹输入: (batch, 50, 9)  轨迹特征序列
    ↓
┌─────────────────────────────────────┐
│      轨迹Bi-LSTM编码器           │
│  • input_dim = 9                  │
│  • hidden_dim = 128               │
│  • num_layers = 2                 │
│  • bidirectional = True           │
└─────────────────┬───────────────┘
                  │
                  ▼
         轨迹表示 (batch, 256)

KG输入: (batch, 50, 11)  KG特征序列
    ↓
┌─────────────────────────────────────┐
│      KGBi-LSTM编码器           │
│  • input_dim = 11                 │
│  • hidden_dim = 64                │
│  • num_layers = 2                 │
│  • bidirectional = True           │
└─────────────────┬───────────────┘
                  │
                  ▼
         KG表示 (batch, 128)
                  │
                  ▼
┌─────────────────────────────────────┐
│      特征融合层（硬拼接）         │
│  • concat([trajectory, KG])       │
│  • Linear(384, 64)                │

└─────────────────┬───────────────┘
                  │
                  ▼

                  │
                  ▼
         类别logits (batch, 7)
```

## 数据处理

### 数据清洗

采用统一的两阶段数据清洗流程：

**第一阶段：基础预处理**
- 填充缺失特征
- 统一轨迹序列长度（50）
- 快速平滑速度/加速度异常点

**第二阶段：深度清洗**
- 剔除异常轨迹段（速度/加速度/方向过大）
- 插值缺失点，处理时间跳变
- 平滑异常方向（bearing）
- 统一序列长度
- 记录清洗统计信息

### 知识图谱构建

从OSM数据中提取地理语义信息：

1. **道路网络提取**
   - 道路类型分类（primary/secondary/residential等）
   - 道路连通性分析

2. **POI提取**
   - 公交站位置
   - 地铁站位置
   - 关键设施位置

3. **空间语义**
   - 市区/郊区划分
   - 功能区域识别

4. **特征生成**
   - 轨迹点道路类型
   - 距离最近POI的距离
   - 空间语义标签
   - 交通网络特征

### 数据划分

- **训练集**: 70%
- **验证集**: 10%
- **测试集**: 20%
- **划分策略**: 分层采样（stratified sampling）

## 训练配置

### 超参数

| 参数 | 值 | 说明 |
|------|-----|------|
| learning_rate | 5e-5 | AdamW优化器学习率 |
| batch_size | 32 | 批次大小 |
| epochs | 50 | 最大训练轮数 |
| hidden_dim | 128 | LSTM隐藏层维度 |
| num_layers | 2 | LSTM层数 |
| dropout | 0.3 | Dropout比率 |
| weight_decay | 1e-4 | L2正则化 |
| max_grad_norm | 1.0 | 梯度裁剪阈值 |
| cleaning_mode | balanced | 数据清洗模式 |

### 训练策略

- **优化器**: AdamW
- **学习率调度**: ReduceLROnPlateau
- **早停机制**: 验证损失10轮不改善则停止
- **梯度裁剪**: 最大梯度范数为1.0
- **损失函数**: 交叉熵损失（CrossEntropyLoss）

| Subway | +0.3443 | 68.1% | 地铁站POI特征显著改善识别 |
| Bus | +0.1580 | 26.5% | 公交站POI特征提供关键信息 |
| Car & taxi | +0.1380 | 25.4% | 道路类型特征改善识别 |
| Bike | +0.0892 | 13.7% | 道路类型和POI特征辅助识别 |
| Walk | +0.0422 | 5.1% | 基础识别较好，提升有限 |
| Train | +0.1441 | 27.0% | 样本少，但POI特征有帮助 |

## 核心发现

### 1. 知识图谱的关键价值

- **准确率提升**: 69.60% → 79.49%（+9.89%）
- **加权F1提升**: 0.6818 → 0.7916（+0.1098）
- **Subway突破**: F1从0.5059提升至0.8502，提升幅度最大（68.1%）

### 2. 地理语义信息的作用

- **POI特征**: 公交站/地铁站距离是关键特征
- **道路类型**: 道路等级信息帮助区分交通方式
- **空间语义**: 市区/郊区划分提供上下文信息

### 3. 类别识别改善

- **公共交通**: Bus、Subway识别显著改善
- **私人交通**: Car & taxi识别明显提升
- **慢速交通**: Walk、Bike识别有所改善

### 4. 稀有类别改善

- **Train**: F1从0.5333提升至0.6774（+27.0%）
- **Airplane**: 样本仍为0，无法识别

## 结论

Exp2验证了知识图谱特征对交通方式识别的关键价值，准确率从69.60%提升至79.49%（+9.89%）。地理语义信息（POI距离、道路类型、空间语义）显著改善了识别性能，特别是对公共交通类别（Bus、Subway）的识别。该实验证明了多源数据融合的有效性，为后续特征工程和融合策略研究奠定了基础。
