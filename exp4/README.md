# Exp4: 轨迹+知识图谱+天气

## 实验概述

Exp4在Exp3的基础上整合天气特征，验证多模态数据融合的效果。该实验的目标是探索环境信息（天气）对交通方式识别的影响，以及多源数据融合的极限。

## 实验设计

### 特征组合

- **轨迹特征**: 9维
  - speed（速度）
  - acceleration（加速度）
  - distance（累计距离）
  - bearing（方向角）
  - bearing_change（方向变化）
  - speed_change（速度变化）
  - time_interval（时间间隔）
  - distance_change（距离变化）
  - duration（持续时间）

- **知识图谱特征**: 15维
  - 道路类型（primary/secondary/residential等）
  - POI距离（公交站/地铁站距离）
  - 空间语义（是否在市区/郊区）
  - 交通网络（道路连通性）
  - 道路宽度、车道数、限速、交通流量等

- **天气特征**: 12维
  - temp（温度）
  - prcp（降水量）
  - snow（降雪量）
  - wspd（风速）
  - rhum（相对湿度）
  - is_rainy（是否下雨）
  - is_heavy_rain（是否大雨）
  - is_snowy（是否下雪）
  - is_cold（是否寒冷<5°C）
  - is_hot（是否炎热>30°C）
  - is_windy（是否大风）
  - normalized_temp（归一化温度）

### 融合策略

- **硬拼接**: 轨迹+KG+天气独立编码后拼接
- **决策层**: 所有特征直接影响分类决策

## 数据处理

### 数据清洗

采用统一的两阶段数据清洗流程：

**第一阶段：基础预处理**
- 填充缺失特征
- 统一轨迹序列长度（50）
- 快速平滑速度/加速度异常点

**第二阶段：深度清洗**
- 剔除异常轨迹段（速度/加速度/方向过大）
- 插值缺失点，处理时间跳变
- 平滑异常方向（bearing）
- 统一序列长度
- 记录清洗统计信息

### 天气数据处理

从北京天气数据中提取环境信息：

1. **数据加载**
   - 加载2007-2012年小时级天气数据
   - 解析时间戳和气象参数

2. **特征提取**
   - 温度、降水量、降雪量、风速、湿度
   - 派生特征：是否下雨、是否大雪、是否寒冷等

3. **时间对齐**
   - 根据轨迹时间戳匹配天气数据
   - 处理缺失值（使用前后值填充）

4. **特征归一化**
   - 数值特征归一化到[0,1]
   - 二值特征保持0/1

### 降级特征处理

当天气数据缺失时，使用降级策略：

- **完全缺失**: 使用零向量替代天气特征
- **部分缺失**: 使用可用特征，缺失部分用零填充
- **时间不匹配**: 使用最近时间点的天气数据

### 数据划分

- **训练集**: 70%
- **验证集**: 10%
- **测试集**: 20%
- **划分策略**: 分层采样（stratified sampling）

## 训练配置

### 超参数

| 参数 | 值 | 说明 |
|------|-----|------|
| learning_rate | 5e-5 | AdamW优化器学习率 |
| batch_size | 32 | 批次大小 |
| epochs | 50 | 最大训练轮数 |
| hidden_dim | 128 | LSTM隐藏层维度 |
| num_layers | 2 | LSTM层数 |
| dropout | 0.3 | Dropout比率 |
| weight_decay | 1e-4 | L2正则化 |
| max_grad_norm | 1.0 | 梯度裁剪阈值 |
| cleaning_mode | balanced | 数据清洗模式 |

### 训练策略

- **优化器**: AdamW
- **学习率调度**: ReduceLROnPlateau
- **早停机制**: 验证损失10轮不改善则停止
- **梯度裁剪**: 最大梯度范数为1.0
- **损失函数**: 交叉熵损失（CrossEntropyLoss）
- **NaN/Inf检测**: 自动跳过异常批次
- **样本保留**: 不丢弃任何样本，使用降级特征

## 评估结果

### 整体性能

| 准确率 | 74.80% | -3.45% |
| 宏平均F1 | 0.6896 | -0.0410 |
| 加权F1 | 0.7373 | -0.0420 |

### 各类别性能

|---------|-----------|--------|----------|---------|-----------|
| Walk | 0.8690 | 0.8690 | 0.8690 | 1263 | -0.0085 |
| Bike | 0.6899 | 0.6899 | 0.6899 | 433 | -0.0302 |
| Bus | 0.6934 | 0.6934 | 0.6934 | 570 | -0.0397 |
| Car & taxi | 0.6377 | 0.6377 | 0.6377 | 480 | -0.0276 |
| Subway | 0.6614 | 0.6614 | 0.6614 | 352 | -0.1643 |

### 性能下降分析

| 类别 | F1变化 | 变化幅度 | 主要原因 |
|------|--------|---------|---------|
| Walk | -0.0085 | -1.0% | 天气特征引入噪声 |
| Bike | -0.0302 | -4.2% | 天气特征引入噪声 |
| Bus | -0.0397 | -5.4% | 天气特征引入噪声 |
| Car & taxi | -0.0276 | -4.1% | 天气特征引入噪声 |
| Subway | -0.1643 | -19.9% | 天气特征严重干扰 |
| Train | -0.5789 | -100.0% | 样本少，天气特征导致失效 |

## 核心发现

### 1. 天气特征的负面影响

- **准确率下降**: 78.25% → 74.80%（-3.45%）
- **加权F1下降**: 0.7793 → 0.7373（-0.0420）
- **噪声引入**: 天气特征可能引入无关信息

### 2. 稀有类别受影响严重

- **Train**: F1从0.5789降至0.0000（-100%）
- **Subway**: F1从0.8257降至0.6614（-19.9%）
- **样本少**: 稀有类别对噪声更敏感

### 3. 多模态融合的挑战

- **特征维度**: 24维 → 36维
- **模型复杂度**: 显著增加
- **性能影响**: 负面影响

### 4. 降级特征处理

- **缺失值处理**: 零填充可能引入偏差
- **时间不匹配**: 最近时间点可能不准确
- **特征质量**: 天气数据质量参差不齐

## 结论

Exp4验证了多模态数据融合的挑战，准确率从78.25%下降至74.80%（-3.45%）。天气特征的引入并未带来性能提升，反而引入了噪声，特别是对稀有类别（Train、Subway）的识别造成严重影响。该实验揭示了多源数据融合的极限，以及特征质量比特征数量更重要的规律。

## 优化建议

1. **特征选择**: 基于重要性分析选择最优天气特征子集
2. **特征降维**: 使用PCA或特征选择方法减少特征维度
3. **特征融合**: 探索更复杂的融合策略（如注意力机制）
4. **数据质量**: 提高天气数据质量和时间匹配精度
5. **模型集成**: 使用模型集成减少单一模型的噪声影响
