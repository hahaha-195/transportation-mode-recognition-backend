# Exp1: 仅使用GPS轨迹特征

## 实验概述

Exp1是整个项目的基线实验，仅使用GPS轨迹特征进行交通方式识别。该实验的目标是建立性能基准，为后续多源数据融合实验提供对比参考。

## 实验设计

### 特征组合

- **轨迹特征**: 9维
  - speed（速度）
  - acceleration（加速度）
  - distance（累计距离）
  - bearing（方向角）
  - bearing_change（方向变化）
  - speed_change（速度变化）
  - time_interval（时间间隔）
  - distance_change（距离变化）
  - duration（持续时间）

### 融合策略

- **无融合**: 仅使用轨迹特征独立编码
- **决策层**: 仅轨迹特征影响分类决策

## 模型架构

## 数据处理

### 数据清洗

采用统一的两阶段数据清洗流程：

**第一阶段：基础预处理**
- 填充缺失特征
- 统一轨迹序列长度（100）
- 快速平滑速度/加速度异常点

**第二阶段：深度清洗**
- 剔除异常轨迹段（速度/加速度/方向过大）
- 插值缺失点，处理时间跳变
- 平滑异常方向（bearing）
- 统一序列长度
- 记录清洗统计信息

### 数据划分

- **训练集**: 70%
- **验证集**: 10%
- **测试集**: 20%
- **划分策略**: 分层采样（stratified sampling）

## 训练配置

### 超参数

| 参数 | 值 | 说明 |
|------|-----|------|
| learning_rate | 5e-5 | AdamW优化器学习率 |
| batch_size | 32 | 批次大小 |
| epochs | 50 | 最大训练轮数 |
| hidden_dim | 128 | LSTM隐藏层维度 |
| num_layers | 2 | LSTM层数 |
| dropout | 0.3 | Dropout比率 |
| weight_decay | 1e-4 | L2正则化 |
| max_grad_norm | 1.0 | 梯度裁剪阈值 |
| cleaning_mode | balanced | 数据清洗模式 |

### 训练策略

- **优化器**: AdamW
- **学习率调度**: ReduceLROnPlateau
- **早停机制**: 验证损失10轮不改善则停止
- **梯度裁剪**: 最大梯度范数为1.0
- **损失函数**: 交叉熵损失（CrossEntropyLoss）

## 核心发现

### 1. 基线性能

- **整体准确率**: 69.60%
- **主要类别**: Walk、Bike、Bus识别效果较好
- **稀有类别**: Train、Airplane样本过少，无法有效识别

### 2. 类别不平衡

- **Walk**: 样本最多（1263），F1最高（0.8319）
- **Airplane**: 样本为0，无法识别
- **Train**: 样本极少（10），识别困难

### 3. 混淆模式

- **Walk vs Bike**: 容易混淆，运动模式相似
- **Bus vs Subway**: 容易混淆，都是公共交通
- **Car & taxi**: 容易与其他类别混淆，特征不明显

## 结论

Exp1作为基线实验，验证了仅使用GPS轨迹特征进行交通方式识别的可行性，准确率达到69.60%。该实验为后续多源数据融合实验提供了性能基准，证明了轨迹特征的有效性，但也暴露了类别不平衡和稀有类别识别困难的问题。
