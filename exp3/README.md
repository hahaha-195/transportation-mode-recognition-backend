# Exp3: 轨迹+增强知识图谱

## 实验概述

Exp3在Exp2的基础上扩展知识图谱特征维度，从11维增加到15维，探索特征工程的边际效应。该实验的目标是验证特征增加是否一定能带来性能提升，以及是否存在特征冗余问题。

## 实验设计

### 特征组合

- **轨迹特征**: 9维
  - speed（速度）
  - acceleration（加速度）
  - distance（累计距离）
  - bearing（方向角）
  - bearing_change（方向变化）
  - speed_change（速度变化）
  - time_interval（时间间隔）
  - distance_change（距离变化）
  - duration（持续时间）

- **增强知识图谱特征**: 15维（vs Exp2的11维）
  - 道路类型（primary/secondary/residential等）
  - POI距离（公交站/地铁站距离）
  - 空间语义（是否在市区/郊区）
  - 交通网络（道路连通性）
  - **新增**: 道路宽度、车道数、限速、交通流量等

### 融合策略

- **硬拼接**: 轨迹+增强KG独立编码后拼接
- **决策层**: 所有特征直接影响分类决策

## 模型架构

### Bi-LSTM模型（多输入）

```

增强KG输入: (batch, 50, 15)  增强KG特征序列
    ↓

                  │

                  │
                  ▼
```

## 数据处理

### 数据清洗

采用统一的两阶段数据清洗流程：

**第一阶段：基础预处理**
- 填充缺失特征
- 统一轨迹序列长度（50）
- 快速平滑速度/加速度异常点

**第二阶段：深度清洗**
- 剔除异常轨迹段（速度/加速度/方向过大）
- 插值缺失点，处理时间跳变
- 平滑异常方向（bearing）
- 统一序列长度
- 记录清洗统计信息

### 增强知识图谱构建

在Exp2基础上扩展特征：

1. **基础特征（与Exp2相同）**
   - 道路类型分类
   - POI距离（公交站/地铁站）
   - 空间语义（市区/郊区）
   - 交通网络（道路连通性）

2. **新增特征**
   - 道路宽度
   - 车道数
   - 限速信息
   - 交通流量
   - 道路坡度
   - 路口密度
   - 建筑密度
   - 功能区域类型

3. **特征选择**
   - 相关性分析
   - 特征重要性排序
   - 冗余特征剔除

### 数据划分

- **训练集**: 70%
- **验证集**: 10%
- **测试集**: 20%
- **划分策略**: 分层采样（stratified sampling）

## 训练配置

### 超参数

| 参数 | 值 | 说明 |
|------|-----|------|
| learning_rate | 5e-5 | AdamW优化器学习率 |
| batch_size | 32 | 批次大小 |
| epochs | 50 | 最大训练轮数 |
| hidden_dim | 128 | LSTM隐藏层维度 |
| num_layers | 2 | LSTM层数 |
| dropout | 0.3 | Dropout比率 |
| weight_decay | 1e-4 | L2正则化 |
| max_grad_norm | 1.0 | 梯度裁剪阈值 |
| cleaning_mode | balanced | 数据清洗模式 |

### 训练策略

- **优化器**: AdamW
- **学习率调度**: ReduceLROnPlateau
- **早停机制**: 验证损失10轮不改善则停止
- **梯度裁剪**: 最大梯度范数为1.0
- **损失函数**: 交叉熵损失（CrossEntropyLoss）

## 核心发现

### 1. 特征工程的边际递减

- **准确率下降**: 79.49% → 78.25%（-1.24%）
- **加权F1下降**: 0.7916 → 0.7793（-0.0123）
- **特征冗余**: 新增特征可能引入冗余信息

### 2. 过拟合问题

- **训练集表现**: 可能优于Exp2
- **测试集表现**: 劣于Exp2
- **泛化能力下降**: 特征增加导致模型复杂度提升

### 3. 类别性能差异

- **Walk**: 唯一提升的类别（+0.0034）
- **其他类别**: 全部下降
- **Train**: 下降最严重（-0.0985）

### 4. 特征选择的重要性

- **特征数量**: 11维 → 15维
- **性能影响**: 负面影响
- **结论**: 特征增加不一定带来性能提升

## 结论

Exp3揭示了特征工程中的边际递减规律，准确率从79.49%下降至78.25%（-1.24%）。特征增加（11维→15维）并未带来性能提升，反而引入了特征冗余和过拟合问题。该实验证明了特征选择的重要性，以及"更多特征不一定更好"的规律，为后续特征工程和模型优化提供了重要启示。

## 优化建议

1. **特征选择**: 基于重要性分析选择最优特征子集
2. **特征降维**: 使用PCA或特征选择方法减少特征维度
3. **正则化增强**: 增加Dropout或L2正则化防止过拟合
4. **早停策略**: 更严格的早停机制
5. **特征工程**: 关注特征质量而非数量
